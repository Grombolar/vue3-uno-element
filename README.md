## Git 提交规范

使用 husky 和 link-staged 规范 git 提交

当前使用了谷歌推广的 angular 项目 git 提交规范，具体参考如下

- `feat` 增加新功能
- `fix` 修复问题/BUG
- `style` 代码风格相关无影响运行结果的
- `perf` 优化/性能提升
- `refactor` 重构
- `revert` 撤销修改
- `test` 测试相关
- `docs` 文档/注释
- `chore` 依赖更新/脚手架配置修改等
- `workflow` 工作流改进
- `ci` 持续集成
- `types` 类型定义文件更改
- `wip` 开发中

实例：

```bash
git commit -am "feat: XXX"
git push
```
